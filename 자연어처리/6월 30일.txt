사전 기반
분석 감정은 2~7 분류를 사용하지만, 대부분 3분류를 사용해야 오류가 없다.

VOC 감성분석
Encoder는 홀수 3, 5명이 적절 (여기선 3명이라 하기)

충북지역 대표 관광 콘텐츠 전략화
블로그는 긍정일 때가 많기 때문에 부정에 집중해야 한다.

감성점수 = 긍정감정 수 - 부정감정 수
감정점수 > 0 : 긍정, 감정점수 = 0 : 중립, 감정점수 < 0 : 부정
감정점수가 중립인 구간을 두기엔 다양한 경우때문에 무리가 있다.
sigmoid 함수를 적용해 중립 구간을 만들기도 하지만, 0에 근접한 값들이 너무 가파르다.

사전 기반과 모델 기반의 차이
사전 기반 : 예측하고자 하는 열을 사전이랑 비교해서 값을 찾음 / 모델 기반 : 예측하고자 하는 열을 다른 열이랑 비교해서 값을 찾음
도메인이 바뀌면 모델 기반의 정확도가 낮아진다.
사전 기반과 모델 기반 중 어떤 것이 좋은지는 확인해봐야 안다.

모델기반
모든 문서에서 나타나는 어휘를 모으고, 일정 빈도 이상으로 컬럼을 만든다. (300000을 기준으로 하면 계산이 너무 힘들어지고 훈련이 부정확해진다. 10000이면 대충 정확)
칼럼에 있는 어휘의 빈도를 기록한다.
각 문서의 종류를 기록하여 훈련 데이터를 생성한다.

딥러닝 : 은닉층이 2개 이상인 모델
tensorflow를 keras가 포함한 형태
(60000, 28, 28) : 28 * 28이 60000개 존재
딥러닝의 단점 : 적절한 데이터가 없으면 확률의 문제이기 때문에 부적절한 결과가 나올 수 있다.

데이터에서는 데이터의 순수 모습이 아닌 훈련 특성이 몇 차원인지에 따라 몇 D인지 나눈다. (ex. mnist data -> 2D Tensor)

딥러닝 감성분석 : Tokenizing -> Padding -> One-Hot Encoding
